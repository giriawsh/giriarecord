# 前言

《JS权威指南》第四章第二部分，涉及运算符（优先级、结合性等）、算术表达式（基本运算符加减乘除等，位运算符）、关系表达式（==和===，比较运算符，in和instanceof）

# 运算符

大多数运算符是由标点符号表示的：+和=等

另外一些是由关键字表示的：delete和instanceof等

下表是运算符优先级，前面的优先级高于后面的优先级，剩下一些参数详情见书本P65

# 操作数个数

大多数运算符是二元运算符，也支持一些一元运算符（-等），也支持三元运算符（?:）

# 操作数类型和结果类型

一些运算符可以作用于任何数据类型，但仍希望它们的操作数是指定类型的数据（可详见上表）。

JS运算符通常会根据需要对操作数进行类型转换。

举例来说

  • 乘法运算符*会将操作数转换为数字。

  • 那些希望操作数是布尔类型的操作符，它们的操作数可以是任意类型（因为JS中所有值不是真值就是假值）

有一些操作符对操作数类型有着不同程度的依赖。

举例来说

  • +可以对数字进行加法，也可以对字符串作连接
  • <比较运算符可以对数字进行大小值的比较，也可以比较字符在字母表中的次序先后

# 左值

左值：指的是表达式只能出现在赋值运算符的左侧

在JS中，变量、对象属性和数组元素均是左值。

ECMA规范允许内置函数返回一个左值，但自定义的函数则不能返回左值。

# 运算符的副作用

有一些表达式具有副作用，前后的表达式运算会互相影响。

举例来说

  • ++和--包含隐式的赋值
  • delete运算符：删除一个属性就像（但不完全一样）给这个属性赋值undefined

其他JS运算符都没有副作用，如果任何用在函数调用表达式和对象创建表达式中的运算符带有副作用，函数调用表达式和对象创建表达式将会带有副作用。（可以理解成每次调用或者创建时，副作用都会随着表达式的使用而发挥作用）

# 运算符优先级

详情可见上表。

  • 优先级高的运算符的执行总是先于优先级低的运算符
  • 运算符的优先级可以通过显式使用圆括号来重写
  • 赋值运算的优先级非常低，通常总是最后执行的

# 基本运算符

  • *、/、%、-在必要的时候将操作数转为数字。

所有那些无法转换为数字的都转换成NaN，算数运算结果也是NaN

  • 在JS中，所有数字都是浮点型的，默认得到的除法运算结果也是浮点型。

    • 0/0结果是NaN
    • 所有这些运算均不会报错

  • %结果的符号和第一个操作数（被除数）的符号保持一致。

# "+"运算符

加号的转换规则优先考虑字符串连接。

如果两个操作数都不是类字符串（字符串或者转换为字符串的对象）的，那么都将进行算术加法运算

运算规则如下：

  • 如果其中一个操作数是对象，对象会遵循对象到原始值的转换规则转换为原始值
    • 日期通过toString()，其他对象通过valueOf()
      • 如果valueOf方法不返回原始值，其他对象也采用toString()
  • 对象转换为原始值后，如果其中一个操作数是字符串，另一个操作数也转换为字符串，然后进行字符串连接
  • 否则，两个操作数都将转换为数字（或者NaN），然后进行加法操作

这里1+{} === 1[object Object]的原因是{}.valueOf() = {}并不是原始值，于是调用的是toString()函数

  • 当加号运算符和字符串和数字一起使用时，需要考虑加法的结合性（从左到右）对运算顺序的影响。

# 一元算术运算符

在JS中，一元运算符作用域一个单独的操作数，并产生一个新值，具有很高的优先级，而且都是右结合。

必要时，他们会将操作数转换为数字。

# 一元加法+

把操作数转换为数字或者NaN,并返回这个转换后的数字

# 一元减法-

把操作数转换为数字，并改变运算结果的符号

# 递增++

对操作数进行增量加一操作，操作数是一个左值（变量、数组元素或对象属性）。

运算符将操作数转换为数字，然后给数字加一，并将加一后的数值重新赋值。

  • ++x称为前增量运算符，返回计算后的值
  • x++称为后增量，返回未做增量计算的值

需要注意的是“++”从不进行字符串连接操作，它总是会将操作数转换为数字并增一。

++x与x=x+1的区别

var x = "1";
++x // => 2
x+1 // => 11

# 位运算符

位运算符可以对由数字表示的二进制数据进行更低层级的按位运算。

位运算符要求它的操作数是整数，这些整数表示为32位整数而不是64位浮点型。

  • 位运算符首先将操作数转换为数字（必要时），并将数字强制表示为32位整数、
    • 这会忽略原格式中的小数部分和任何超过32位的二进制位。

注意

  • 移位运算符要求右操作数在0~31之间，在其操作数转换为无符号32位整数后，右操作数将舍弃第5位之后的二进制位（相当于超过31的数字将会进行mod32的操作），以便生成一个位数正确的数字（0~31）

  • 位运算符会将NaN、Infinity和-Infinity都转换为0

# 按位非~

将操作数的所有为取反

根据JS中带符号的整数的表示方法，对一个值使用~运算符相当于改变它的符号并减一。

# 左移<<

将操作数所有的二进制位数进行左移，移动的位数是0~31之间的一个整数（超过将进行mod32运算）

在数字没有溢出的前提下，对于正数和负数，左移一位都相当于乘以2的1次方，左移n位就相当于乘以2的n次方。

# 带符号右移>>

将一个操作数的所有位进行右移操作，移动的位数是0~31之间的一个整数（超过将进行mod32运算）

右边溢出的位将忽略，填补在左边的位由原操作数的符号决定，正数补零，负数补1，以便保持结果的符号与原操作数一直。

将一个值右移1位，相当于用它除以2（忽略余数），以此类推

# 无符号右移>>>

与>>一样，只是左边的高位总是填补0，与原来的操作数符号无关。

运用移位符号进行整数转换

var signed = -1;
var unsigned = signed >>> 0;     // >>> 0 转换为无符号数
console.log(unsigned);                 // 4294967295
signed = unsigned << 0;          // << 0 转换为有符号数
console.log(signed);                   // -1

# 溢出相关

在 JavaScript 中，所有整数字变量默认都是有符号整数。

有符号整数使用 31 位表示整数的数值，用第 32 位表示整数的符号，0 表示正数，1 表示负数。 数值范围从 - 2^31 ~ 2^31 - 1 即 - 2147483648 到 2147483647。

对于 Javascript 移位操作来说，默认的左移 << 和右移 >> 操作都是有符号数移位操作，于是有了二进制表示第 31 位为 1 的整数左移后结果成了负数、二进制表示第 32 位为 1 的整数右移后结果成了负数的情况

alert(2147483647<<1);   // -2 
alert(4294967294>>1);   // -1

# instanceof运算符

instanceof运算符希望左操作数是一个对象，右操作数标识对象的类。

如果左侧对象是右侧类的实例，则表达式返回true，否则返回false。

由于JS中对象的类是通过初始化它们的构造函数来定义的，所以也可以说instanceof的右操作数应当是一个函数。

特殊处理

如果左操作数不是对象的话，instanceof返回false，如果右操作数不是函数，则会抛出一个类型错误异常。

TIP

所有的对象都是Object的实例。

当通过instanceof判断一个对象是否是一个类的实例的时候，这个判断也会包含对“父类”的检测。

instanceof的工作原理将会涉及“原型链”，JS的继承机制等。后面将会进行详细描述。

为了计算表达式o instanceof f， JS会首先计算f.prototype，然后在原型链中查找o。如果找到，那么o是f（或者f的父类）的一个实例，表达式返回true，如果f.prototype不在o的原型链中，那么o就不是f的实例，instanceof返回false。（？翻译好绕，我真的没看懂……等第六章以后再来回顾吧）
