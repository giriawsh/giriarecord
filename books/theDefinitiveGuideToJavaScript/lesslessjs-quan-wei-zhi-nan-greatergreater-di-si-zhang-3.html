# 逻辑表达式

# 逻辑与&&

&&运算符可以从三个不同的层次进行理解

  1. 当操作数都是布尔值时，&&对两个值执行AND操作。返回true或false

关系运算符的优先级比&&和||要高

  1. &&可以对真值和假值（false, null, undefined, 0, -0, NaN, ""）进行AND操作。返回真值或假值

在JS中任何希望使用布尔值的地方，表达式和语句都会将其当做真值或假值来对待。

  1. 运算符首先计算左操作数的值（即&&左侧的表达式），如果是假值，&&此时简单地返回左操作数的值；如果左操作数是真值，&&将计算右操作数的值并将其返回作为整个表达式的计算结果。
var o = { x: 1 };
var p = null;
o && o.x // => 1 左值是真值，计算了右值后返回
p && p.x // => null

&&的行为有时被称作短路

if (a==b) stop()
等价于
(a==b) && stop()

当&&右侧的表达式具有副作用（赋值、递增、递减、函数调用表达式）时，要小心，因为这些表达式的执行依赖于左操作数的计算结果。

右侧有副作用应尽量避免

var a = 0;
a++ && a // => 0 因为a++返回0，是个假值

var b = 1;
b++ && b// => 2 b++返回1，是个真值，返回右值

var c = 2;
c && c++ // => 2 c++返回2
c // => 3

var d = 3;
d && (b = d + 2) // => 5 赋值返回计算后结果

var e = 4;
(e = e -3) && function(){return --e;}// => ƒ (){return --e;} 函数调用返回函数调用

# 逻辑或||

大多数情况下知识做简单OR运算。

首先计算第一个操作数的值，如果是真值，返回这个真值；否则，计算第二个操作数的值，并返回这个表达式的计算结果。

同样应当避免右操作数包含一些具有副作用的表达式

TIP

这个运算符最常用的方式是用来从一组备选表达式中选出第一个真值表达式。

这种用法通常用在函数体内，用来给参数提供默认值

# 逻辑非！

一元运算符，目的是将操作数的布尔值进行求反。

！首先将其操作数转换为布尔值（&&和||并不会这么做），然后再对布尔值求反。

！ 总是返回true或者false

TIP

可以通过两次逻辑非运算来得到一个值的等价布尔值：!!x

！具有很高的优先级

有这样两个恒等式：

!(p&&q) === !p || !q
!(p||q) === !p && !q

# 赋值表达式

JS使用=来给变量或者属性赋值。=运算符希望它的左操作数是一个左值（一个变量或者对象属性），它的右操作数可以是任意类型的任意值。

赋值表达式的副作用时：右操作数的值赋值给左侧的变量或者对象属性，后续对这个变量和对象属性的引用都将得到这个值。

=具有非常低的优先级。

赋值操作符的结合性是从右至左。因此可以这样给多个变量赋值：

i=j=k=0;
1

# 带操作的赋值运算

就是这些东西：

在大多数情况下：

a op= b
等价于
a = a op b
1
2
3

第一行a被计算了一次，第二行a计算了两次。

只有在a包含具有副作用的表达式（比如函数调用和赋值操作）的时候，两者才不等价。

举个例子：

data[i++] *= 2;
data[i++] = data[i++] * 2

# 表达式计算

JS可以解释运行由JS源代码组成的字符串，并产生一个值。JS通过全局函数eval()来完成这个工作。

eval("3+2") // => 5
1

TIP

动态判断源代码中的字符串是一种强大的语言特性，几乎没有必要在实际中应用。

如果使用了eval，应当仔细考虑是否真的需要。

# eval()基本用法

eval()只有一个参数

  • 如果传入的参数不是字符串，它直接返回这个参数
  • 如果传入字符串，它会把字符串当成JS代码进行编译。
    • 如果编译失败，抛出一个语法错误异常
      • 这个异常将把调用传递给eval()
    • 如果编译成功，则开始执行这段代码，并返回空字符串中的最后一个表达式或语句的值
      • 如果最后一个表达式或语句没有值，则最终返回undefined

关于eval()最重要的是，它使用了调用它的变量作用域环境。

var x = 1;
eval("y = x + 1")
y // => 2

# 全局eval()

eval()具有更改局部变量的能力，这对于JS优化器来说是一个很大的问题。

为了让JS解释器的实现更加简化，ECMAScript3规定任何解释器都不允许对eval赋予别名。否则抛出一个EvalError异常。

实际上，大多数实现中，当通过别名调用时，eval()会将其字符串当成顶层的全局代码来执行。

当成顶层的全局代码执行不会影响函数内的代码优化

执行的代码可能会定义新的全局变量和全局函数，或者给全局变量赋值。

但是却不能使用或修改主调函数中的局部变量。

因此，这不会影响到函数内的代码优化。

ECMAScript5是反对使用EvalError的，并且规范了eval()的行为。

直接eval

当直接使用非限定的eval名称来调用eval()函数时，被称为直接eval。

（就是不给它重新赋值给别的名字。）

var f = eval;//使用了别名调用的eval
var g = f;
1
2

  • 直接调用eval()时，它总是在调用它的上下文作用域内执行。

  • 其他的间接调用则使用全局对象作为其上下文作用域，并且无法读、写、定义局部变量和函数。

如果是全局的话，在外层console x或者y时会得到修改，否侧不会。

全局eval是一种很有用的特性，它允许我们执行那些对上下文没有任何依赖的全局脚本代码段（其实是一件很神奇的事，明明在函数内部使用，却最终影响的只有全局）

# 严格eval()

ECMAScript5严格模式下调用eval()时，eval执行的代码段可以查询或更改局部变量，但不能在局部作用域中定义新的变量或函数。

且将“eval”列为保留字，不能用一个别名覆盖eval()函数。变量名、函数名、函数参数或者异常捕获的参数都不能取名为eval。

（由于无法进行间接赋值，自然不可能对全局产生影响）

# 总结：

ECMAScript3

  • 规定任何解释器都不允许对eval赋予别名。否则抛出一个EvalError异常。

  • 大多数实现中，当通过别名调用时，eval()会将其字符串当成顶层的全局代码来执行。

  • 可能会定义新的全局变量和全局函数，或者给全局变量赋值。

  • 不能使用或修改主调函数中的局部变量。

ECMAScript5

  • 反对使用EvalError。

  • 直接调用eval()时，它总是在调用它的上下文作用域内执行。

  • 其他的间接调用则使用全局对象作为其上下文作用域，并且无法读、写、定义局部变量和函数。

ECMAScript5严格模式

  • eval执行的代码段可以查询或更改局部变量，但不能在局部作用域中定义新的变量或函数。

  • 将“eval”列为保留字，任何东西都不能取名为eval

  • 由于无法进行间接赋值，自然不可能对全局产生影响

# 其他运算符

# 条件运算符?:

唯一一个三元运算符。操作数可以是任意类型。

典型场景，判断一个变量是否有定义（并拥有一个有意义的真值）。

# typeof运算符

一元运算符，操作数可以是任意类型。

返回值 表示操作数类型的一个字符串。

typeof运算符可以带上圆括号，让typeof看起来像一个函数名，而不是一个运算符关键字：

typeof(i);
1

如果想将null和普通对象区分开，必须针对特殊值显式检测。

对于宿主对象来说，typeof可能并不返回object，而返回字符串。

但实际上客户端JS中的大多数宿主对象都是object类型。

对于所有对象和数组的typeof结果都是object，因此它对于区分对象和其他原始值来说是很有帮助的。

区分方法

  • typeof

区分对象和其他原始值

  • instanceof、class特性、constructor属性

区别对象的类

尽管JS中的函数是对象的一种，但typeof还是将特殊特殊对待，返回function。

在JS中，函数和“可执行的对象”(callable object)有着微妙的区别。

  • 所有函数都是可执行的。
  • 对象也有可能是可执行的，可以像调用函数一样调用它，但是它并不是一个真正的函数。

ES3规范：对于所有内置可执行对象，typeof返回function

ES5规范：所有可执行对象（包括内置对象和宿主对象），typeof返回function

大多数浏览器厂商都将JS的原生函数对象当成它们的宿主对象。

# *内置对象、宿主对象、原生对象

JS标准内置对象清单

  1. 内置（Build-in）对象与原生（Naitve）对象的区别在于：前者总是在引擎初始化阶段就被创建好的对象，是后者的一个子集；而后者包括了一些在运行过程中动态创建的对象。

  2. 宿主对象不是引擎的原生对象，而是由宿主框架通过某种机制注册到JavaScript引擎中的对象。

  3. 一些宿主会把自己提供的对象／构造器也称为“原生对象”，例如Internet Explorer 7就把它提供的XMLHttpRequest()称为原生的——与此相对的是在它的更早先版本中通过“new ActiveXObject('Microsoft.XMLHTTP')”这样的方法创建的对象。这种情况下，读者应注意到“宿主的原生对象”与“引擎的原生对象”之间的差异。

# 一、原生对象：

ECMA-262 把原生对象（native object）定义为“独立于宿主环境的 ECMAScript 实现提供的对象”。

Object、Function、Array、String、Boolean、Number、Date、RegExp、Error、EvalError、RangeError、ReferenceError、SyntaxError、TypeError、URIError、ActiveXObject(服务器方面)、Enumerator(集合遍历类)、RegExp（正则表达式）

由此可以看出，简单来说，原生对象就是 ECMA-262 定义的类（引用类型）。

# 二、内置对象

ECMA-262 把内置对象（built-in object）定义为“由 ECMAScript 实现提供的、独立于宿主环境的所有对象，在 ECMAScript 程序开始执行时出现”。这意味着开发者不必明确实例化内置对象，它已被实例化了。

每个内置对象都是原生对象。

# 三、宿主对象

何为“宿主对象”？ ECMAScript中的“宿主”当然就是我们网页的运行环境，即“操作系统”和“浏览器”。所有非原生对象都是宿主对象（host object），即由 ECMAScript 实现的宿主环境提供的对象。

所有的 BOM 和 DOM 对象都是宿主对象。因为其对于不同的“宿主”环境所展示的内容不同。其实说白了就是，ECMAScript官方未定义的对象都属于宿主对象，因为其未定义的对象大多数是自己通过ECMAScript程序创建的对象。

# delete运算符

delete是一元操作符，它用来删除对象属性或者数组元素。

var o = {x:1, y:2};
delete o.x;
"x" in o; // => false

删除属性或者数组元素不仅仅是设置了一个undefined的值。

删除后这个元素不存在了，读取一个不存在的值将返回undefined，但是可以通过in运算符来检测这个属性是否在对象中存在。

delete希望操作数是一个左值。

  • 不是左值，delete不进行任何操作，且返回true
  • 是左值，delete将试图删除这个指定的左值
    • 删除成功，delete返回true（包括删除不存在的属性）

一些不可删除的属性：

  1. 一些内置核心和客户端属性
  2. 用户通过var语句声明的变量不能删除
  3. 通过function语句定义的函数和函数参数。

ES5严格模式中，如果delete的操作数是非法的（例如变量、函数、函数参数），delete操作将抛出一个语法错误。

只有操作数是一个属性访问表达式的时候它才会正常工作。

delete删除不可配置属性:

  • 严格模式下，会抛出一个类型错误异常
  • 非严格模式下，不会报错，只是简单地返回false，以表明操作数不能执行删除操作。

# void运算符

void是一元运算符,它出现在操作数之前，操作数可以是任意类型。

操作数会照常计算，但忽略计算结果并返回undefined。

TIP

由于void会忽略操作数的值，因此在操作数具有副作用的时候使用void来让程序更具语义。

这个运算符最常用在客户端的URL——javascript:URL中。在URL中可以写带有副作用的表达式，而void则让浏览器不必显示这个表达式的计算结果。

例如经常在a标签中使用的void运算符：

<a href="javascript:void window.open();">打开一个新窗口</a>
1

当然也可以通过给a标签的onclick绑定一个事件处理程序。

# 逗号运算符

逗号运算符是二元运算符，操作室可以是任意类型。

它首先计算左操作数，然后右操作数，最后返回 右操作数的值。

总是会计算左侧的表达式，但计算结果会被忽略。

只有左侧表达式更具有副作用，才会使用逗号运算符让代码变得更通顺。

最常用的地方当然是在for循环里了。

for(var i = 0, j = 10; i < j; i++, j--);
